# Product

## Register

product

## Users

One keyboard-first macOS power user (and people like them). Mid-task when they summon it — muscle memory, eyes barely leave their work. The palette appears over anything: any wallpaper, any Space, full-screen apps, light or dark system appearance. Ambient context is unpredictable, so the surface must read everywhere.

## Product Purpose

The fastest possible local launcher: press a shortcut, type, open or execute. Success = sub-perceptible latency, near-zero background footprint, and being invisible until summoned. Not a Raycast clone — a small, strong feature set (apps, calculator, opt-in file search, clipboard history, user scripts) done precisely.

## Brand Personality

Instant, native, restrained. Playful name, polished tool — the interface itself is quiet and disappears into the task. Reference: Raycast's *craft* (material quality, spacing discipline, footer action hints), not its chrome verbatim.

## Anti-references

- Raycast feature-copy or visual clone — borrow the quality bar, keep our own identity.
- Web-app-in-a-window: custom scrollbars, invented form controls, themed chrome.
- Feature-heavy launcher UIs with badges, tips, and promos crowding the list.
- Decorative motion of any kind; a launcher that animates is a launcher that's slow.

## Design Principles

1. **Disappear into the task** — the palette is system furniture, not an app vying for attention.
2. **Earned familiarity** — system materials, dynamic system colors, SF Symbols; never invented affordances for standard actions.
3. **Keyboard is the interface** — every visual element must serve or explain a keystroke; pointer is optional.
4. **State is always visible** — active mode, primary-action hint, permission/indexing notes; no hidden modes.
5. **Restraint** — one accent (the system's), density over decoration, nothing on screen that doesn't earn its row.

## Accessibility & Inclusion

VoiceOver labels on rows, query field, and footer hints. Text contrast ≥ 4.5:1 including placeholder text, on both appearances over the translucent material. No motion, so nothing to reduce. Respects system appearance automatically (no forced dark).
